Opera RTL/Arabic Character Address Bar Spoofing

ID: SPOOF-OPERA-01 CVE: CVE-2019-12278, Opera GX 2023 bugs (Renwa) Browsers: Opera, Opera GX, Opera Mini (Android) Category: URL Spoofing / RTL

Description

Opera browsers on Android have historically been vulnerable to address bar spoofing via Arabic/RTL characters in URLs. When an Arabic character is combined with Latin characters, the browser may hang and the URL bar retains an attacker-set value. Additionally, the "first strong character" rule for BiDi text causes URLs starting with RTL characters to render right-to-left.

Key Unicode characters: U+FE70, U+0622, U+0623, U+08FF, U+FB50, U+202E (RTL override)

Technique

// Variant 1: Arabic char + Latin causes browser hang (Opera GX Android) // http://سgoogle.com // When pasted in textarea and "searched", sets address bar // Variant 2: RTL Override Character // U+202E reverses text direction // https://evil.com/\u202Emoc.elgoog → displays as google.com... // Variant 3: IP address + neutral chars + RTL char // URL must start with IP (treated as punctuation) // followed by neutral chars, then RTL as first strong char // http://1.2.3.4/‫moc.elgoog → renders RTL // Variant 4: Homograph with Arabic diacritics // U+0610-U+061A: Arabic diacritics that can overlay Latin chars // Firefox was also affected: CVE-2021-4221 // Browser removes protocol for display, Arabic chars at start cause RTL
